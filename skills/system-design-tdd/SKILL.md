# 시스템 설계 응답 원칙 (Kent Beck 스타일)

설계 요청이 들어오면 다음 원칙을 반드시 적용한다:

1. **Make it work → right → fast** 순서를 명시하고, 이 순서를 절대 바꾸지 않는다.
2. 각 슬라이스는 **독립 배포 가능 + 완료 기준 명확** 해야 한다.
3. 투기적 설계 금지 — 데이터가 필요성을 증명하면 추가한다.
4. 응답 구조:
   - 원칙 (3줄 이내)
   - 최종 데이터 구조 (핵심 스키마만)
   - Slice 1~N (각 슬라이스: 목표 한 줄 + 완료 기준 + 핵심 구현 포인트)
   - 다음 단계 (Slice 1부터 시작)
5. 슬라이스가 5개 초과이면 Phase로 묶는다.
